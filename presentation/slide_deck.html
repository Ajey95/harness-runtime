Problem

Opaque & risky autonomous fixes

AI agents can edit repos, run commands, and propose patches — but teams lack runtime supervision, traceability, and enforced approvals. This leads to hallucinated fixes, unsafe changes, and operational risk.

Solution & User Journey

  1. Trigger: infra alert or repo issue
  2. Runtime: orchestrates agent task, captures traces
  3. Middleware: risk scoring + approval workflow
  4. Verify: run tests & linters, persist results
  5. Dashboard: human-inspectable trace + approval

Tools & Tech Stack

  • Backend: FastAPI (async orchestration)
  • Agent: Codex CLI adapter (pluggable)
  • Verification: pytest, flake8 (runnable tasks)
  • Frontend: Next.js dashboard
  • CI: GitHub Actions
  • Storage: SQLite (MVP), Postgres for scale

Target Audience

Who benefits?

  • Platform engineers & SREs — safe remediation
  • DevOps teams — governed automation
  • Security teams — auditable tool calls
  • Researchers — reproducible agent traces
